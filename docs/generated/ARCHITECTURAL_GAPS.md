# ARCHITECTURAL GAPS

This document explicitly catalogues the discrepancies between the "desired state" (as written in the `README.md`) and the "actual state" (the codebase).

| Subsystem | Stated Design | Actual Codebase Reality | Severity |
|---|---|---|---|
| **Authentication** | `auth-service` providing JWTs via Spring Security | No `auth-service` directory. No `spring-security` dependencies. Unauthenticated API Gateway. | 🔴 CRITICAL |
| **Anomaly Detection** | Statistical Z-Score 1-hr baselines emitted to `anomaly.detected` | Alerts are generated by simply checking if a trace contains an `ERROR` flag. The `anomaly-detection-service` does not exist. | 🔴 CRITICAL |
| **Resilience** | Resilience4j circuit breakers on BFF protecting against LLM failure | Standard WebClient with a silent `onErrorResume` returning empty data. No state machine. | 🟠 HIGH |
| **RAG / Vectors** | `rag-service` using `pgvector` for semantic search on postmortems | No `rag-service`. No vector models or SQL extensions in the AI DB. | 🟠 HIGH |
| **Observability** | OpenTelemetry tracing and Azure App Insights | Standard Spring Boot Actuator. No OpenTelemetry javaagents or exporters configured in any POM. | 🟠 HIGH |
| **CI/CD & Cloud** | GitHub Actions deploying via Bicep to Azure Container Apps | `infrastructure` and `.github` folders do not exist. Only local Docker Compose available. | 🟠 HIGH |
| **Tests** | Comprehensive enterprise-grade testing | `trace-service` and `bff-service` have exactly zero automated tests. | 🔴 CRITICAL |
